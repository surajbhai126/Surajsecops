ARP (Address Resolution Protocol)

Learn ARP from Beginner to Advanced

ARP stands for Address Resolution Protocol. It is used in IPv4 networks to find the MAC address associated with a known IP address.

Why is ARP Needed?

Devices on a network use IP addresses for logical communication and MAC addresses for local network communication.

When a device knows the destination IP address but does not know the destination MAC address, ARP is used to discover the MAC address.

Example of ARP

Suppose a computer wants to communicate with:

IP Address: 192.168.1.20

But the computer does not know the MAC address of that device.

The computer sends an ARP Request asking:

"Who has 192.168.1.20?"

The device with that IP address responds with its MAC address.

How ARP Works

Step 1: ARP Request

The source device broadcasts an ARP Request to all devices on the local network.

Example:

Who has 192.168.1.20?

Step 2: ARP Reply

The device that owns the requested IP address sends an ARP Reply containing its MAC address.

Example:

192.168.1.20 is at AA:BB:CC:11:22:33

Step 3: ARP Cache

The source device stores the IP-to-MAC mapping in its ARP cache.

ARP Request vs ARP Reply

ARP Request ARP Reply
Usually broadcast Usually unicast
Asks for a MAC address Provides a MAC address
Sent to multiple devices Sent to the requesting device

What is an ARP Cache?

An ARP cache is a temporary table that stores IP address and MAC address mappings.

Example:

IP Address MAC Address
192.168.1.1 AA:BB:CC:11:22:33
192.168.1.20 DD:EE:FF:44:55:66

How to View ARP Cache

Windows

Use the following command:

arp -a

Linux

Common commands include:

ip neigh

or:

arp -a

What is Gratuitous ARP?

Gratuitous ARP is an ARP message sent by a device without receiving an ARP Request first.

It can be used to:

  • Update ARP caches
  • Detect duplicate IP addresses
  • Update network devices after an IP change

What is ARP Spoofing?

ARP Spoofing is a network attack in which an attacker sends false ARP messages to associate their MAC address with another device's IP address.

This can allow attackers to redirect network traffic through an unauthorized device.

ARP Poisoning and Cybersecurity

ARP Poisoning can potentially enable Man-in-the-Middle attacks on a local network.

Security professionals monitor unusual ARP activity to detect suspicious network behavior.

ARP Security and Prevention

  • Dynamic ARP Inspection (DAI)
  • DHCP Snooping
  • Static ARP Entries where appropriate
  • Network segmentation
  • Monitoring unusual ARP traffic

ARP and the OSI Model

ARP operates between the Network Layer and Data Link Layer concepts.

It helps map a Layer 3 IPv4 address to a Layer 2 MAC address.

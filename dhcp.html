DHCP Explained: Complete Beginner Guide

Learn Dynamic Host Configuration Protocol from Beginner to Advanced

DHCP stands for Dynamic Host Configuration Protocol. It is a network protocol that automatically assigns IP configuration to devices on a network.

What is DHCP?

When a device connects to a network, it needs important network information to communicate with other devices.

This information may include:

  • IP Address
  • Subnet Mask
  • Default Gateway
  • DNS Server

DHCP automatically provides this information to network devices.

Why is DHCP Used?

Without DHCP, network administrators would need to manually configure every device.

DHCP makes network management easier by automatically assigning network configuration.

  • Reduces manual configuration
  • Prevents many IP address conflicts
  • Simplifies network management
  • Automatically provides DNS and gateway information

DHCP Components

DHCP Client

A DHCP client is a device that requests network configuration.

Examples include:

  • Computer
  • Smartphone
  • Printer
  • Network device

DHCP Server

A DHCP server assigns IP addresses and other network configuration to DHCP clients.

DHCP Relay Agent

A DHCP Relay Agent forwards DHCP messages between clients and servers located on different networks.

DHCP DORA Process

DHCP uses a four-step process known as DORA.

DORA = Discover → Offer → Request → Acknowledgement

1. DHCP Discover

The client broadcasts a DHCP Discover message to find available DHCP servers.

2. DHCP Offer

The DHCP server responds with an offer containing an available IP address and network configuration.

3. DHCP Request

The client sends a DHCP Request message to request the offered IP configuration.

4. DHCP Acknowledgement

The DHCP server sends a DHCP Acknowledgement and officially assigns the configuration to the client.

DHCP Ports

Device Port
DHCP Server UDP 67
DHCP Client UDP 68

What is a DHCP Lease?

A DHCP lease is the amount of time for which a client can use an assigned IP address.

When the lease expires, the client may request a renewal from the DHCP server.

What is DHCP Reservation?

DHCP reservation allows a specific device to receive the same IP address every time.

The reservation is usually created using the device's MAC address.

DHCP reservation is useful for devices such as:

  • Printers
  • Servers
  • Network devices
  • Security cameras

What is DHCP Relay?

DHCP broadcasts normally do not cross routers.

If the DHCP server is located on a different network, a DHCP Relay Agent can forward DHCP requests to the DHCP server.

This allows one centralized DHCP server to provide IP addresses to multiple networks.

DHCP with VLANs

In networks with multiple VLANs, DHCP Relay is commonly used.

For example:

  • VLAN 10 – Sales
  • VLAN 20 – HR
  • VLAN 30 – IT

A centralized DHCP server can provide IP addresses to all these VLANs using DHCP Relay.

DHCP and Cybersecurity

DHCP is important in cybersecurity because attackers can abuse DHCP services.

Network security professionals monitor DHCP activity to identify suspicious behavior.

What is DHCP Starvation Attack?

A DHCP Starvation attack attempts to consume all available IP addresses from a DHCP server.

When the available IP pool is exhausted, legitimate devices may not receive an IP address.

Network security controls such as switch security features can help reduce this risk.

What is a Rogue DHCP Server?

A Rogue DHCP Server is an unauthorized DHCP server connected to a network.

It may provide incorrect network configuration to clients.

This can potentially redirect network traffic or cause connectivity problems.

What is DHCP Snooping?

DHCP Snooping is a security feature available on many managed network switches.

It helps identify trusted and untrusted DHCP traffic.

DHCP Snooping can help protect networks against unauthorized DHCP servers.

Common DHCP Troubleshooting

If a device cannot connect to a network, DHCP may be one of the areas to investigate.

Common checks include:

  • Is the DHCP server running?
  • Is the DHCP address pool exhausted?
  • Is the client connected to the correct VLAN?
  • Is DHCP Relay configured correctly?
  • Is the network interface working properly?

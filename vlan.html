VLAN Explained: Complete Guide to VLANs, Access Ports, Trunk Ports & 802.1Q

Learn VLANs from Beginner to Advanced

VLAN stands for Virtual Local Area Network. A VLAN allows a physical network to be logically divided into multiple separate networks.

VLANs are widely used in enterprise networking, cybersecurity and network segmentation.

What is a VLAN?

A VLAN is a logical network created on a physical network infrastructure.

A single physical switch can be divided into multiple logical networks using VLANs.

For example:

  • VLAN 10 – Sales
  • VLAN 20 – HR
  • VLAN 30 – IT
  • VLAN 40 – Management

Even if all devices are connected to the same physical switch, VLANs logically separate their networks.

Why are VLANs Used?

  • Network segmentation
  • Improved network security
  • Reduced broadcast traffic
  • Better network management
  • Logical separation of departments
  • Improved network performance

VLAN and Broadcast Domain

A broadcast domain is a group of devices that receive broadcast traffic from each other.

Each VLAN represents a separate broadcast domain.

For example:

VLAN 10 ≠ VLAN 20

Devices in different VLANs cannot communicate directly at Layer 2. Communication between different VLANs requires a Layer 3 device.

What is a VLAN ID?

A VLAN ID is a numerical identifier used to identify a specific VLAN on a network.

Each VLAN is assigned a unique VLAN ID.

Example:

  • VLAN 10 – Sales
  • VLAN 20 – HR
  • VLAN 30 – IT

Switches use VLAN IDs to logically separate network traffic.

Types of VLANs

1. Default VLAN

The default VLAN is usually VLAN 1. Switch ports are commonly assigned to this VLAN by default.

2. Data VLAN

A Data VLAN carries normal user traffic such as web browsing, file transfers and application traffic.

3. Voice VLAN

A Voice VLAN is used for VoIP phones. It helps separate voice traffic from normal data traffic.

4. Management VLAN

A Management VLAN is used to manage network devices such as switches, routers and firewalls.

5. Native VLAN

The Native VLAN carries untagged traffic over an 802.1Q trunk link.

What is an Access Port?

An access port is normally assigned to a single VLAN.

Access ports are commonly used to connect end devices such as:

  • Computers
  • Printers
  • Servers
  • IP Phones

For example, a computer connected to an access port assigned to VLAN 10 belongs to VLAN 10.

What is a Trunk Port?

A trunk port carries traffic from multiple VLANs over a single physical link.

Trunk ports are commonly used between:

  • Switch to Switch
  • Switch to Router
  • Switch to Layer 3 Switch

Access Port vs Trunk Port

Access Port Trunk Port
Carries traffic for one VLAN Carries traffic for multiple VLANs
Usually connects end devices Usually connects network devices
Frames are normally untagged VLAN tags identify VLAN traffic

What is VLAN Tagging?

VLAN tagging is the process of adding VLAN identification information to an Ethernet frame.

The most commonly used VLAN tagging standard is:

IEEE 802.1Q

VLAN tags allow switches to identify which VLAN a frame belongs to.

What is IEEE 802.1Q?

IEEE 802.1Q is a networking standard used for VLAN tagging on Ethernet networks.

It allows multiple VLANs to travel across a single trunk link.

The VLAN tag contains information that helps network devices identify the VLAN associated with a frame.

What is Inter-VLAN Routing?

Devices in different VLANs cannot communicate directly at Layer 2.

Inter-VLAN routing allows communication between different VLANs using a Layer 3 device.

Inter-VLAN routing can be performed using:

  • Router-on-a-Stick
  • Layer 3 Switch

VLANs and Network Security

VLANs are important for network security because they help segment networks.

Common security zones may include:

  • User VLAN
  • Server VLAN
  • Guest VLAN
  • Management VLAN

Firewalls and access control policies can be applied between different network segments.

What is VLAN Hopping?

VLAN hopping is a network attack where an attacker attempts to access traffic belonging to another VLAN.

Proper switch configuration and security controls can help reduce the risk of VLAN hopping.

VLAN Security Best Practices

  • Disable unused switch ports
  • Avoid using the default VLAN for management
  • Configure trunk ports carefully
  • Use dedicated management VLANs
  • Allow only required VLANs on trunk links

Practical VLAN Example

Imagine an organization with three departments:

Department VLAN ID Network Purpose
Sales VLAN 10 Sales users
HR VLAN 20 HR users
IT VLAN 30 IT users

Each department can be logically separated even if all devices are connected to the same physical switch.

Conclusion

VLANs are an essential part of modern networking. They allow organizations to logically divide networks, reduce broadcast traffic and improve security.

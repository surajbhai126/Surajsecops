Common Network Protocols and Ports

Learn Network Protocols and Ports from Beginner to Advanced

Network protocols are rules and standards that allow devices to communicate with each other over a network.

A port number identifies a specific service or application running on a device.

What is a Network Protocol?

A network protocol is a set of rules that defines how data is transmitted and received between devices.

Examples of network protocols include:

  • HTTP
  • HTTPS
  • DNS
  • SSH
  • FTP
  • SMTP
  • DHCP
  • ICMP

What is a Port?

A port is a logical communication endpoint used to identify a specific service or application.

For example, a web server commonly uses:

Port 80 → HTTP

Port 443 → HTTPS

TCP and UDP Ports

Ports can use different transport layer protocols, mainly TCP or UDP.

TCP UDP
Reliable communication Faster communication
Connection-oriented Connectionless
Uses acknowledgements No delivery guarantee

HTTP - Port 80

HTTP stands for Hypertext Transfer Protocol. It is used to transfer web pages and other web resources between a client and a web server.

Default Port: 80/TCP

HTTPS - Port 443

HTTPS is the secure version of HTTP. It uses encryption to protect data exchanged between the client and server.

Default Port: 443/TCP

DNS - Port 53

DNS converts domain names into IP addresses.

Example:

google.com → IP Address

DNS commonly uses:

  • UDP Port 53 for normal queries
  • TCP Port 53 for specific operations

SSH - Port 22

SSH stands for Secure Shell. It provides secure remote access to systems using encrypted communication.

Default Port: 22/TCP

FTP - Port 21

FTP stands for File Transfer Protocol. It is used to transfer files between systems.

FTP is not encrypted by default.

Default Control Port: 21/TCP

DHCP - Ports 67 and 68

DHCP automatically assigns network configuration to devices.

DHCP uses UDP:

  • UDP 67 → DHCP Server
  • UDP 68 → DHCP Client

SMTP - Port 25

SMTP stands for Simple Mail Transfer Protocol. It is used for sending email messages.

Default Port: 25/TCP

RDP - Port 3389

RDP stands for Remote Desktop Protocol. It allows users to remotely access a Windows system.

Default Port: 3389/TCP and UDP

Why are Ports Important in Cybersecurity?

Open ports can expose services running on a system. Attackers may scan systems to identify open ports and vulnerable services.

Security professionals use port scanning during authorized security assessments to identify exposed services.

Common Port Scanning Concepts

  • Open Port
  • Closed Port
  • Filtered Port
  • Service Detection
  • Version Detection

Network Protocols and Firewalls

Firewalls can control network traffic based on IP addresses, ports and protocols.

For example, a firewall rule may allow:

HTTPS → TCP → Port 443

and block unnecessary services such as insecure or unused ports.

Conclusion

Network protocols define how devices communicate, while port numbers identify specific services running on those devices.

Understanding common protocols and ports is essential for networking, troubleshooting, firewalls, penetration testing and cybersecurity.

Subnetting Explained: CIDR, Subnet Mask & Practical Examples

Learn Subnetting from Beginner to Advanced

Subnetting is one of the most important concepts in computer networking and cybersecurity. It allows a large IP network to be divided into smaller logical networks called subnets.

What is Subnetting?

Subnetting is the process of dividing a large IP network into smaller networks. These smaller networks are called subnets.

For example:

192.168.1.0/24

This network can be divided into multiple smaller networks according to the organization's requirements.

Subnetting helps organizations efficiently use IP addresses, improve network management and separate different departments or security zones.

Why is Subnetting Important?

  • Efficient use of IP addresses
  • Reduces unnecessary broadcast traffic
  • Improves network performance
  • Improves network security
  • Makes network management easier
  • Helps separate different departments

IPv4 Address Structure

An IPv4 address contains 32 bits. These 32 bits are divided into four 8-bit octets.

Example:

192.168.1.10

Each octet contains 8 bits.

Therefore:

8 + 8 + 8 + 8 = 32 bits

Each octet can contain a value from:

0 to 255

What is a Subnet Mask?

A subnet mask is used to identify which portion of an IP address represents the network and which portion represents the host.

Example:

IP Address: 192.168.1.10

Subnet Mask: 255.255.255.0

In this example:

  • 192.168.1 represents the Network Portion
  • 10 represents the Host Portion

What is CIDR Notation?

CIDR stands for Classless Inter-Domain Routing .

CIDR notation represents the number of network bits using a slash followed by a number.

Example:

192.168.1.0/24

The /24 means that the first 24 bits are used for the network portion.

The remaining 8 bits are available for hosts.

Common Subnet Masks

CIDR Subnet Mask Total Addresses Usable Hosts
/24 255.255.255.0 256 254
/25 255.255.255.128 128 126
/26 255.255.255.192 64 62
/27 255.255.255.224 32 30
/28 255.255.255.240 16 14

How to Calculate Usable Hosts?

For normal IPv4 subnets, the formula is:

Usable Hosts = 2ⁿ − 2

Here, n represents the number of host bits.

We subtract 2 because:

  • One address is reserved for the Network Address
  • One address is reserved for the Broadcast Address

Note: /31 and /32 networks are special cases and are commonly used for point-to-point links and host routes.

Example: 192.168.1.0/24

A /24 network has:

  • 24 Network Bits
  • 8 Host Bits

Total Addresses:

2⁸ = 256 Addresses

Usable Hosts:

256 − 2 = 254 Usable Hosts

Network Address:

192.168.1.0

Usable Host Range:

192.168.1.1 – 192.168.1.254

Broadcast Address:

192.168.1.255

Practical Example: Dividing a /24 Network

Suppose we have:

192.168.1.0/24

If we divide this network into /26 subnets, we get four smaller networks.

Subnet Network Address Usable Host Range Broadcast Address
1 192.168.1.0/26 192.168.1.1 - 192.168.1.62 192.168.1.63
2 192.168.1.64/26 192.168.1.65 - 192.168.1.126 192.168.1.127
3 192.168.1.128/26 192.168.1.129 - 192.168.1.190 192.168.1.191
4 192.168.1.192/26 192.168.1.193 - 192.168.1.254 192.168.1.255

What is VLSM?

VLSM stands for Variable Length Subnet Mask .

VLSM allows different subnets to use different subnet masks according to their requirements.

For example, suppose an organization has:

  • IT Department: 30 Hosts
  • Sales Department: 10 Hosts
  • HR Department: 5 Hosts

Instead of giving every department the same size network, VLSM allows IP addresses to be allocated more efficiently.

Department Required Hosts CIDR Usable Hosts
IT 30 /27 30
Sales 10 /28 14
HR 5 /29 6

Subnetting and Cybersecurity

Subnetting is also important for cybersecurity. Organizations can use separate subnets for different departments and security zones.

For example:

  • User Network
  • Server Network
  • Guest Network
  • Management Network

Firewalls and Access Control Lists can then be used to control traffic between different network segments.

Common Subnetting Mistakes

  • Assigning the Network Address to a host
  • Assigning the Broadcast Address to a host
  • Using an incorrect subnet mask
  • Calculating the wrong subnet range
  • Confusing total addresses with usable hosts

Conclusion

Subnetting is an essential networking skill. It helps divide large networks into smaller, efficient and manageable networks.

To master subnetting, practice calculating:

  • Network Address
  • Broadcast Address
  • Usable Host Range
  • Number of Subnets
  • Number of Usable Hosts

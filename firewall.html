What is a Firewall?

Learn Firewalls from Beginner to Advanced

A firewall is a security device or software that monitors and controls incoming and outgoing network traffic based on predefined security rules.

In simple words, a firewall acts as a security barrier between a trusted network and an untrusted network.

Simple Firewall Example

Consider the following network:

Internet → Firewall → Internal Network

The firewall examines network traffic and decides whether the traffic should be allowed or blocked.

Why is a Firewall Needed?

Without proper security controls, unauthorized traffic may reach internal systems and services.

A firewall helps organizations:

  • Block unauthorized network traffic.
  • Allow legitimate communication.
  • Control access to network services.
  • Protect internal networks.
  • Monitor network connections.

How Does a Firewall Work?

A firewall analyzes network traffic and compares it with configured security rules.

Based on the rules, the firewall may:

  • Allow the traffic.
  • Block the traffic.
  • Log the traffic.
  • Inspect the traffic further.

Firewall Rule Example

Source Destination Service Action
Internal Network Internet HTTPS Allow
Unknown Network Internal Server SSH Deny

Important Firewall Rule Components

Firewall rules commonly use several parameters to control network traffic.

  • Source: The origin of the traffic.
  • Destination: The target system or network.
  • Source Port: The port from which traffic originates.
  • Destination Port: The service port being accessed.
  • Protocol: TCP, UDP, ICMP and other protocols.
  • Action: Allow, Deny or Drop.

Types of Firewalls

1. Packet Filtering Firewall

A packet filtering firewall examines individual network packets.

It can filter traffic based on:

  • Source IP address
  • Destination IP address
  • Port number
  • Protocol

2. Stateful Firewall

A stateful firewall tracks the state of active network connections.

It understands whether a packet belongs to an existing legitimate connection.

3. Proxy Firewall

A proxy firewall acts as an intermediary between the client and the destination server.

The client does not directly communicate with the destination server.

4. Next-Generation Firewall (NGFW)

A Next-Generation Firewall provides advanced traffic inspection and security capabilities.

Common features may include:

  • Application identification
  • User identification
  • Intrusion prevention
  • Advanced traffic inspection
  • Threat detection

Network Firewall vs Host-Based Firewall

Network Firewall Host-Based Firewall
Protects multiple systems or networks. Protects an individual device.
Usually placed at network boundaries. Installed directly on the endpoint.

Firewall: Allow, Deny and Drop

Allow

The firewall permits the traffic to continue to its destination.

Deny

The firewall rejects the traffic according to the configured rule.

Drop

The firewall silently discards the traffic without sending a response.

Inbound and Outbound Traffic

Inbound Traffic

Inbound traffic enters a network or device from an external source.

Outbound Traffic

Outbound traffic leaves a network or device toward an external destination.

Firewall and Ports

Firewalls can control access to specific ports.

For example:

Allow TCP Port 443 → HTTPS

Block TCP Port 23 → Telnet

This allows organizations to control which services are accessible.

Firewall and Applications

Modern firewalls can identify applications instead of relying only on port numbers.

For example, a firewall may identify a specific application using traffic analysis and apply security policies accordingly.

Firewall and Network Security

Firewalls are an important part of network security architecture.

However, a firewall alone cannot provide complete security.

Organizations should also use:

  • Strong authentication
  • Network segmentation
  • Endpoint security
  • Intrusion detection and prevention
  • Security monitoring
  • Regular updates

Firewall in Cybersecurity

Cybersecurity professionals use firewalls to control network access and reduce unauthorized communication.

Security analysts may investigate firewall logs to identify suspicious connections, blocked traffic and unusual activity.
